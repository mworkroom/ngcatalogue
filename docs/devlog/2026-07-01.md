# 2026-07-01 ngcatalogue Development Log

## Work Summary

기존 단일 `index.html` 기반 가격표 앱을 React + TypeScript + Vite 구조로 전환했다. 화면 디자인과 공개 가격표 기능은 유지하되, Supabase 연결값은 코드에서 분리해 Vite 환경 변수로 관리하도록 바꾸고, PWA 자산은 Vite의 공개 자산 구조에 맞게 `public/` 아래로 이전했다.

## Work Completed

* React, TypeScript, Vite 기반 프로젝트 설정을 추가하고 `npm run dev`, `npm run build`, `npm run preview` 스크립트를 구성했다.
* 기존 HTML 안에 있던 UI와 상태 관리를 `App`, 언어 전환, 검색창, 테이블, 모바일 카드, 배지, 로딩/오류 표시 컴포넌트로 분리했다.
* `PublicProduct` 타입을 추가하고 공개 View `catalog_public_products`에서 `id`, `name_ko`, `name_pt`, `business_price`, `consumer_price`, `brazil_price`, `is_set` 컬럼을 조회하도록 정리했다.
* Supabase URL과 Publishable Key를 `import.meta.env.VITE_SUPABASE_URL`, `import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY`로 읽도록 변경했다.
* 기존 로컬 연결이 끊기지 않도록 현재 공개 Supabase 설정은 `.env.local`에 보존하고, 공유용 `.env.example`에는 빈 키를 제공했다.
* 검색은 한국어명과 포르투갈어명을 모두 대상으로 하며, 공백과 대소문자, 포르투갈어 악센트를 무시하도록 유지했다.
* 검색어가 있을 때만 세트 상품을 먼저 정렬하고, 검색어가 없을 때는 현재 언어의 상품명 기준으로 정렬하도록 구현했다.
* 모바일 카드의 터치 하이라이트와 `summary` 포커스 스타일을 조정해 파란 터치 잔상이 생기지 않도록 했다.
* 기존 `manifest.webmanifest`, `sw.js`, `icons/`를 `public/` 아래로 옮겨 Vite에서 `/manifest.webmanifest`, `/sw.js`, `/icons/...` 경로로 제공되도록 했다.
* 서비스 워커는 Supabase 같은 외부 API 요청을 처리하지 않고, Vite 빌드 산출물인 `assets/` 파일은 런타임 캐시에 저장하도록 보완했다.

## Decisions

* Supabase Publishable Key는 브라우저에서 사용 가능한 공개 키지만, 코드에 직접 남기지 않고 환경 변수로 옮겼다. 이렇게 하면 이후 GitHub Pages나 다른 배포 환경에서도 환경별 설정을 분리할 수 있다.
* 이번 범위는 기존 사업자용 공개 가격표 화면 유지와 React 전환에 집중했다. 관리자 화면, 센터 화면, PDF 직접 생성, 완전한 오프라인 데이터 캐시는 아직 구현하지 않았다.
* GitHub Pages의 하위 경로 배포 가능성을 고려해 Vite `base`는 `./`로 설정했다. 매니페스트와 아이콘도 기존 상대 경로를 유지해 저장소 하위 경로에서도 깨질 가능성을 낮췄다.
* React 개발 모드에서 Supabase 요청이 중복될 수 있는 상황을 피하기 위해 `StrictMode`는 사용하지 않았다. 현재 앱은 단순 조회 화면이고, 반복 요청을 줄이는 것이 기존 동작 유지에 더 적합하다고 판단했다.

## Work Summary

2단계 작업으로 센터 사용자가 공유 8자리 접근 코드로 전체 센터용 상품 정보를 읽을 수 있는 `/catalog/center` 화면을 추가했다. 공개 카탈로그 `/catalog`는 기존처럼 `catalog_public_products`만 조회하도록 유지하고, 센터 전용 데이터는 Supabase Edge Function `catalog-center`를 통해서만 가져오도록 서버 경계를 분리했다.

## Work Completed

* `catalog-center` Edge Function을 추가해 `login`, `products` action을 처리하도록 구현했다.
* `login`은 `CENTER_ACCESS_CODE`와 입력값을 비교하고, 성공 시 `CENTER_SESSION_SECRET`으로 서명한 30일 만료 JWT를 발급한다.
* `products`는 토큰 서명, 만료 시간, `role: "center"`를 검증한 뒤 서비스 역할 키로 `catalog_products`에서 `is_visible = true` 상품만 조회한다.
* 프론트엔드에 `catalog_center_session` 저장소 키를 도입하고, 접근 코드 자체는 저장하지 않으며 토큰과 만료 시각만 저장하도록 했다.
* 카탈로그 표시 컴포넌트를 `public`과 `center` 모드로 나누어 공용 테이블, 모바일 카드, 검색, 정렬, 상품명 fallback 로직을 재사용했다.
* 센터 모드에서 취급 수수료, 브라질 사업자/소비자/사이트 가격, 브라질 PV, 한국 PV, 한국 가격, 중량, 메모, 업데이트 날짜를 표시하도록 컬럼을 확장했다.
* 센터 기본 언어는 포르투갈어로 두고, 한국어 전환은 상품명, 라벨, 검색/상태 문구, 정렬 locale에만 영향을 주도록 유지했다.
* 모바일 센터 화면은 접힌 상태에서 상품명만 보여주고, 펼친 상태에서 센터 컬럼 전체를 순서대로 보여주도록 변경했다.
* 센터 출력은 A4 가로 방향의 named page 규칙을 사용하고, 검색/언어/로그아웃/상태 UI가 출력되지 않도록 공개 출력과 분리했다.
* README를 새로 추가해 프론트 환경 변수, Supabase secrets, Edge Function 배포 절차를 문서화했다.

## Decisions

* 센터 사용자는 개인 계정이 필요 없는 운영 흐름이므로 공유 8자리 코드를 사용하되, 코드가 짧은 점을 고려해 실제 코드는 Edge Function secret으로만 보관하고 잘못된 로그인에는 짧은 지연을 넣었다.
* 공개 앱에 서비스 역할 키나 센터 접근 코드를 넣지 않기 위해 센터 상품 조회는 Edge Function 내부의 service role 클라이언트로만 수행한다.
* 세션은 자동 갱신 없이 30일 만료로 정했다. 센터 사용자가 반복 로그인 부담을 줄일 수 있으면서도 영구 토큰이 남지 않도록 하기 위한 절충이다.
* 센터와 공개 카탈로그는 같은 검색/정렬/표시 컴포넌트를 공유하되, 모드별 컬럼 설정만 다르게 두었다. 이 방식이 공개 화면에 private 컬럼이 섞일 위험을 줄이고 이후 관리자 모드 확장에도 유리하다고 판단했다.
* `updated_at`은 반환 필드에 포함되어 있으므로 센터 화면에도 표시했다. `id`는 데이터 식별과 React key에 사용하지만 사용자가 읽는 컬럼으로는 노출하지 않았다.
* `.env.example`에는 프론트에서 안전한 `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`만 유지했다. `CENTER_ACCESS_CODE`, `CENTER_SESSION_SECRET`, `SUPABASE_SERVICE_ROLE_KEY`는 README의 수동 설정 항목으로만 기록했다.
